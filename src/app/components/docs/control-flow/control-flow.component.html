Description

Angular uses built-in control flow blocks, which tell the framework when and how your templates should be rendered. Examples of this include: displaying content based on a certain condition, rendering a list of items based on application data, etc.

Conditional Rendering

@if @else

One of the most common scenarios that developers encounter is the desire to show or hide content in templates based on a condition.

Rendering lists

@for @empty

Another common scenario developers encounter is the need to render a list of items.

You can optionally include an @empty section immediately after the @for block content. The content of the @empty block displays when there are no items.

track property

When using @for to render a list of items, you can use the track property to improve performance. The track property takes a unique identifier for each item in the list. This allows Angular to track which items have changed, and only update those items. When in doubt, using track $index is a good default.

This ensures any updates to the list are reflected correctly in the UI and tracked properly within Angular, especially in the case of stateful elements or animations.

Implicit variables that are available in a @for
Variable Description
$index The index of the current item in the list
$count The total number of items in the list
$even True if the current item has an even index
$odd True if the current item has an odd index
$first True if the current item is the first item in the list
$last True if the current item is the last item in the list
